NRZ-L (Non-Return to Zero - Level)

Encoding Rules

  • Bit 1: Represented by high voltage (+V)
  • Bit 0: Represented by low voltage (-V)
  • Signal level directly corresponds to bit value
  • No transition occurs in the middle of a bit interval
  • Transition only occurs when bit value changes
‚úì Advantages
  • Simple implementation
  • Easy to generate and decode
  • Efficient bandwidth usage
‚úó Disadvantages
  • No self-clocking capability
  • DC component present
  • Sync issues with long runs
Simple 2-Level

NRZ-I (Non-Return to Zero - Inverted)

Encoding Rules

  • Bit 1: Causes a transition at the beginning of the interval
  • Bit 0: No transition occurs (signal stays at current level)
  • Also known as NRZI or differential encoding
  • Polarity independent - inverting signal doesn't affect decoding
‚úì Advantages
  • Polarity independent
  • Better sync on 1s
  • Simple differential encoding
‚úó Disadvantages
  • Long runs of 0s cause sync loss
  • Still has DC component
  • Not self-clocking
USB Differential

Bipolar AMI (Alternate Mark Inversion)

Encoding Rules

  • Bit 0: Represented by zero voltage (0V)
  • Bit 1: Alternates between positive (+V) and negative (-V) voltage
  • First 1 is +V, next 1 is -V, then +V, and so on
  • Uses three voltage levels: +V, 0, -V
  • Violation of alternation indicates error
‚úì Advantages
  • No DC component
  • Error detection capability
  • Lower bandwidth than NRZ
‚úó Disadvantages
  • Long 0s cause sync loss
  • Requires 3 signal levels
  • Not self-clocking
Telecom T1/E1 3-Level

Pseudoternary

Encoding Rules

  • Bit 1: Represented by zero voltage (0V)
  • Bit 0: Alternates between positive (+V) and negative (-V) voltage
  • Exact opposite of Bipolar AMI
  • First 0 is +V, next 0 is -V, then +V, and so on
  • Uses three voltage levels: +V, 0, -V
‚úì Advantages
  • No DC component
  • Same advantages as AMI
  • Error detection possible
‚úó Disadvantages
  • Long 1s cause sync loss
  • Requires 3 signal levels
  • Not commonly used
ISDN 3-Level

Manchester Encoding

Encoding Rules (G.E. Thomas Convention)

  • Bit 1: Low-to-High transition in the middle of bit interval
  • Bit 0: High-to-Low transition in the middle of bit interval
  • Every bit has a transition in the middle
  • Additional transitions may occur at bit boundaries
  • Transition direction determines bit value
‚úì Advantages
  • Self-clocking
  • No DC component
  • Easy error detection
  • Always synchronized
‚úó Disadvantages
  • Requires 2x bandwidth
  • More complex circuitry
  • Less efficient
Ethernet 10BASE-T Self-Clocking

Differential Manchester

Encoding Rules

  • Always: Transition in the middle of every bit interval
  • Bit 0: Transition at the beginning of bit interval
  • Bit 1: No transition at the beginning of bit interval
  • Mid-bit transition is for clocking only
  • Start-of-bit transition (or lack thereof) carries data
‚úì Advantages
  • Self-clocking
  • Polarity independent
  • Excellent noise immunity
  • Reliable synchronization
‚úó Disadvantages
  • Requires 2x bandwidth
  • Complex implementation
  • More transitions needed
Token Ring Self-Clocking Robust

üìä When to Use Each Encoding

Use NRZ-L/NRZ-I when:

  • Simplicity is priority
  • External clock is available
  • Short distance transmission
  • Bandwidth efficiency needed

Use Bipolar AMI when:

  • DC balance is required
  • Error detection is needed
  • Long-distance transmission
  • Telephone systems (T1/E1)

Use Manchester when:

  • Self-clocking is essential
  • Ethernet networks (10Mbps)
  • Reliable sync required
  • Error detection important

Use Diff. Manchester when:

  • Maximum noise immunity needed
  • Polarity independence required
  • Token Ring networks
  • High reliability environments
