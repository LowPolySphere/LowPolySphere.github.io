G.E. Thomas Convention (Used Here)

  • Bit 1: Low-to-High transition (‚Üë) in the middle of bit interval
  • Bit 0: High-to-Low transition (‚Üì) in the middle of bit interval

Note: IEEE 802.3 uses the opposite convention. This visualizer follows the G.E. Thomas convention where 1 = rising edge.

Key Characteristics

  • Every bit period has exactly one transition in the middle
  • Additional transitions occur at bit boundaries when consecutive bits are the same
  • Transition direction (not just presence) determines bit value
  • Requires twice the bandwidth of NRZ

‚ö° Properties

‚úì Advantages
  • Self-clocking - clock embedded in signal
  • No DC component - average voltage is zero
  • Easy synchronization recovery
  • Error detection - missing transitions indicate errors
  • Works without separate clock line
‚úó Disadvantages
  • Requires 2x bandwidth of NRZ
  • More complex encoding/decoding
  • Maximum modulation rate = 2x bit rate
  • Less efficient for high-speed applications

üîß Applications

Ethernet (10BASE-T)

Original 10 Mbps Ethernet uses Manchester encoding. Each bit requires two signal elements, resulting in a 20 MHz signaling rate.

RFID Systems

Many RFID protocols use Manchester encoding for reliable data transfer between tags and readers.

Magnetic Storage

Historical use in magnetic tape and early disk storage systems due to self-clocking property.

IR Communication

Some infrared communication protocols use Manchester encoding for robust signal transmission.
